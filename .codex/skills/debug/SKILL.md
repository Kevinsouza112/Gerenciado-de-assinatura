# Debug

Debug with evidence.

Process:
1. Identify the expected behavior.
2. Identify the actual behavior.
3. Reproduce the issue.
4. Check error messages and logs.
5. Inspect the affected files.
6. Find the smallest likely cause.
7. Apply the smallest fix.
8. Verify the fix.

Do not guess.
Do not rewrite large parts of the system unless necessary.
Prefer small patches.
Explain what changed and why.
