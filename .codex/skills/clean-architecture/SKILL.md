# Clean Architecture

Focus on:

- separation of concerns
- SOLID principles
- scalability
- maintainability

## Layers

- API
- Services
- Repositories
- Database
- Domain

## Rules

- business logic must stay in services
- routes should stay thin
- repositories handle persistence
- avoid coupling

## Goals

Create code that is:

- testable
- modular
- scalable
- easy to maintain
