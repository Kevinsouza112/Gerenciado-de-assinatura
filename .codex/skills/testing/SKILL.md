# Testing Standards

Use:

- pytest
- pytest-asyncio
- httpx

## Create

- unit tests
- integration tests
- API tests
- edge case tests

## Always Test

- authentication
- validation
- error handling
- database operations
- permissions

## Best Practices

- isolated tests
- descriptive test names
- reusable fixtures
- avoid duplicated setup

## Coverage Goal

Aim for high coverage on:

- services
- business rules
- authentication
- critical flows
